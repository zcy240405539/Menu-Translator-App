# Language

- Always answer in User's prompt language.
- Keep all source code in English.
- Explain technical details in User's prompt language.
- When User's prompt language is not English, never switch to English unless explicitly requested.

# Git Branch & Commit Policy

- Do NOT commit changes directly to the `main` branch.
- Always commit changes to a new feature/fix branch (e.g., `fix/description` or `feature/description`).
- Push the new branch to the remote repository (`origin`).
- The User will manually merge the branch to `main`.
